# 总体规划

## 1. 项目目标

本项目是一个云文档在线管理平台，面向学生作业场景设计。目标是在严格遵守原生 JavaWeb 技术边界的前提下，完成一个可以演示、可以答辩、可以继续扩展的基础版本。

项目将参考 `xilixili` 的组织方式与写法，但业务模型将从视频平台切换为文档平台。

本项目固定包含以下核心方向：

- 支持 TEAM 功能
- 一个用户可以加入多个 TEAM
- 知识库支持个人空间与 TEAM 空间
- 文章编辑器采用 Milkdown
- 多人协同采用 Yjs
- 实时通信采用 WebSocket
- 文章页 / 编辑页提供右侧聊天与协作交流侧边栏

---

## 2. 技术约束

### 后端约束

- 使用 Java 语言开发
- 使用 MVC 架构
- 使用 MySQL / MariaDB
- 使用 JDBC 操作数据库
- 使用 Servlet 做原生 JavaWeb 开发
- 使用 Json 作为网络数据传输格式
- 使用 Maven 管理依赖
- 使用 Git 管理版本
- 禁止使用 Spring / SpringBoot / MyBatis / SpringMVC 等高级框架

### 前端约束

- 以前后端分离方式实现
- 页面要求能展示数据并进行简单交互
- 风格参照 `xilixili`
- 不追求复杂视觉设计，优先保证业务链路和演示完整性

---

## 3. 与参考项目的一致性原则

项目组织、架构和具体写法尽量向 `xilixili` 靠拢，主要体现在：

- 后端继续采用 `constant / config / util / po / vo / dao / service / servlet / filter / exception` 分层
- Servlet 继续采用统一基类 + 注解路由分发思路
- 使用统一异常处理与统一响应结构
- 前端继续采用统一布局、统一请求封装、统一路由守卫
- 文档继续采用 `README + notes/架构` 的组织方式

但以下内容必须重新设计：

- 数据库结构
- 业务实体模型
- 页面内容与路由
- 权限关系
- 协同与聊天方案

---

## 4. MVP 范围

### 首批必须完成

- 用户注册、登录、退出、当前用户
- 图片验证码、邮箱验证码、忘记密码
- 个人中心基础能力
- TEAM 创建、查看、加入、退出
- 一个用户加入多个 TEAM
- 个人知识库与 TEAM 知识库
- 知识库文章列表与文章详情
- 文章创建、修改、删除
- 文档聊天侧边栏
- Milkdown 单人编辑
- Yjs + WebSocket 基础协同

### 暂不纳入首版

- 点赞、评论、收藏、关注
- 企业多级部门与复杂 RBAC
- 全局通知中心
- 全文搜索增强
- 历史版本对比恢复
- 音视频或文件消息聊天

---

## 5. 主要挑战

### P1：数据库模型设计

因为存在用户、TEAM、知识库、成员关系、文章、协同房间、聊天消息等实体，若数据模型不稳定，后续 DAO、Service 与前端接口都会反复返工。

### P2：权限边界设计

必须提前明确：

- TEAM 管理权限谁有
- 知识库归属与成员授权如何区分
- 哪些用户能读、写、管理某篇文章

### P3：协同系统落地难度

Yjs、Milkdown 与 WebSocket 的组合会提升开发复杂度，因此必须先完成单人可用，再逐步接入多人协同。

### P4：学生作业时间控制

这个项目天然容易越做越大，因此必须坚持“先完整、再增强”的策略。

---

## 6. 总体策略

### 6.1 先做骨架，再做业务

优先搭建：

- 后端基础层
- 前端基础层
- 数据库脚本
- 文档结构

### 6.2 先做单人功能，再做多人实时功能

实现顺序建议为：

1. 用户与权限
2. TEAM
3. 知识库
4. 文章 CRUD
5. Milkdown 单人编辑
6. WebSocket 聊天
7. Yjs 协同编辑

### 6.3 优先保证答辩可讲解性

所有设计都应尽量清楚、直接、容易解释，而不是为了高级感引入过多难以说明的方案。

- 源代码

- `sql/`：数据库结构与测试数据
- `notes/架构/`：设计文档
